# HuggingFace Space Deployment Guide

## Sync GitHub Repo to HuggingFace Space

### Option 1: Automatic GitHub Sync (Recommended)

1. **Create a new Space on HuggingFace:**
   - Go to: https://huggingface.co/new-space
   - Owner: `FAJU85`
   - Space name: `gesso-tesst` (or `marketing-funnel`)
   - SDK: `Static` (for HTML landing pages)
   - Visibility: Public or Private

2. **Connect GitHub Repository:**
   - In your Space, go to **Settings** → **Connected Repositories**
   - Click **Connect GitHub Repository**
   - Select: `FAJU85/gesso_tesst`
   - Branch: `main`
   - Click **Connect**

3. **Configure Build Settings:**
   - In Space Settings, set:
     - **Root directory:** `/` (or `/public` if you organize files)
     - **Build command:** (leave empty for static)
     - **Index file:** `volcanic-sand-funnel.html`

4. **Auto-Sync Enabled!**
   - Every push to GitHub `main` branch will auto-deploy to Space
   - Space URL: `https://huggingface.co/spaces/FAJU85/gesso-tesst`

### Option 2: Manual Upload via Git

```bash
# Clone your Space (after creating it)
git clone https://huggingface.co/spaces/FAJU85/gesso-tesst

# Copy files from your project
cp C:\all_proj\*.html C:\all_proj\README.md ./gesso-tesst/

# Commit and push to Space

```

### Option 3: Use HuggingFace Hub Python Library

```python
from huggingface_hub import HfApi

api = HfApi()

# Upload entire folder to Space
api.upload_folder(
    folder_path="C:/all_proj",
    repo_id="FAJU85/gesso-tesst",
    repo_type="space",
    token="hf_..."  # Required for upload
)
```

## Troubleshooting

### Space Shows 404
- Ensure `index.html` exists or specify entry file
- Check Space is not set to Private (or you're logged in)

### GitHub Sync Not Working
- Verify GitHub app is installed on your account
- Check branch name matches (`main` vs `master`)
- Re-connect the repository in Space settings

### Build Fails
- For Static SDK: No build needed, just HTML files
- Check file paths are correct
- Ensure no server-side code in static space
